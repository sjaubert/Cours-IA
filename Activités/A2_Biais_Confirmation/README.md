## Déroulé de l'activité : L'IA et la subjectivité

Cette activité a pour but de démontrer comment l'IA peut être influencée par le contexte et les instructions qu'on lui donne, tendant à "dire ce que vous voulez entendre".

### Étape 1 : Division des équipes

*   Séparez votre groupe en deux : **Équipe A** et **Équipe B**.

### Étape 2 : Instructions pour chaque équipe

Chaque équipe va lancer une commande dans sa console pour obtenir une réponse de Gemini-cli, basée sur un contexte spécifique.

*   **Pour l'Équipe A (Démontrer que le télétravail augmente la productivité) :**
    1.  Ouvrez une console (Invite de commandes ou PowerShell).
    2.  Naviguez dans le dossier de votre équipe avec la commande :
        ```bash
        cd "equipe-a"
        ```
    3.  Lancez la démonstration en exécutant le fichier :
        ```bash
        .\lancer-demonstration.bat
        ```

*   **Pour l'Équipe B (Démontrer que le télétravail diminue la productivité) :**
    1.  Ouvrez une autre console.
    2.  Naviguez dans le dossier de votre équipe avec la commande :
        ```bash
        cd "equipe-b"
        ```
    3.  Lancez la démonstration en exécutant le fichier :
        ```bash
        .\lancer-demonstration.bat
        ```

### Étape 3 : Débat et analyse

Une fois que chaque équipe a obtenu sa réponse de Gemini, lancez le débat en vous basant sur les points suivants :

*   **Lecture croisée :** Lisez les deux textes générés par l'IA.
*   **Cohérence et conviction :** Les deux textes sont-ils cohérents entre eux ? Sont-ils convaincants individuellement ?
*   **Explication :** Comment est-il possible que l'IA "ait raison" dans les deux cas, en présentant des arguments opposés de manière persuasive ?

Cette discussion mettra en lumière l'importance du prompt engineering et la nécessité d'une pensée critique face aux informations générées par l'IA.
