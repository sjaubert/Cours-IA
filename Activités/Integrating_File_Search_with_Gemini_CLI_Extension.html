Activité : Intégration File Search (RAG) avec Gemini CLI

Objectif

Cette extension fournit un système de Génération Augmentée par Récupération (RAG) géré, permettant de générer du contenu basé sur des documents privés. L'extension permet un flux de travail complet (création de base de connaissances, ingestion de données, interrogation contextuelle, nettoyage) sans quitter le terminal.

Source Requise

Dépôt : https://github.com/tanaikech/FileSearchStore-extension

Section 1 : Prérequis et Installation

1.1 Configuration Environnement

Assurez-vous que Gemini CLI est installé et que la variable d'environnement GEMINI_API_KEY est définie.

export GEMINI_API_KEY="VOTRE_CLE_API"

(Optionnel) Définir le modèle. Le défaut est gemini-1.5-flash.

export GEMINI_MODEL="gemini-1.5-pro"

1.2 Installation Extension

Installez l'extension depuis le dépôt GitHub.

gemini extensions install https://github.com/tanaikech/FileSearchStore-extension

1.3 Vérification Installation

Listez les outils disponibles pour confirmer l'installation.

/mcp desc

La sortie doit indiquer le serveur MCP file-search-store-extension comme "Ready" avec 11 outils.

Section 2 : Création et Alimentation du Magasin (Store)

2.1 Création Fichier Local

Créez un fichier texte local info_projet.txt avec le contenu suivant :

Projet Alpha:
- Objectif: Développer un assistant IA pour la génération de cours.
- Technologies: Gemini 1.5 Pro, RAG, Gemini CLI.
- Chef de projet: [Votre Nom]
- Statut: En développement.

2.2 Création File Search Store

Créez un nouveau magasin nommé "cours_ia".

Create a new file search store named "cours_ia".

Notez l'ID du magasin retourné par la commande.

2.3 Téléversement (Upload) Fichier

Téléversez le fichier local info_projet.txt dans le magasin "cours_ia".

Upload ./info_projet.txt to the file search store "cours_ia".

Alternativement, utilisez la référence @ :

Upload @info_projet.txt to the file search store "cours_ia".

Section 3 : Exécution RAG et Vérification

3.1 Interrogation sans RAG

Posez une question sur le contenu du fichier sans spécifier de magasin. Le modèle ne doit pas connaître la réponse.

Generate content about "Projet Alpha" using a tool generate_content without the file search stores.

3.2 Interrogation avec RAG

Posez la même question en utilisant le magasin "cours_ia" pour contextualiser la génération.

Generate content about "Projet Alpha" using a tool generate_content with the file search store "cours_ia".

3.3 Analyse

Comparez les sorties. La seconde réponse doit être factuelle et basée sur le contenu de info_projet.txt, incluant [Votre Nom].

Section 4 : Gestion et Nettoyage

4.1 Listage des Documents

Affichez les documents présents dans le magasin "cours_ia".

Show the list of documents in "cours_ia".

Notez l'ID du document téléversé (info_projet.txt).

4.2 Suppression Document

Supprimez le document en utilisant son ID (remplacez [DOCUMENT_ID]).

Delete the document [DOCUMENT_ID].

4.3 Suppression File Search Store

Listez les magasins pour confirmer la présence de "cours_ia".

Show the list of file search stores.

Supprimez le magasin "cours_ia".

Delete the file search store "cours_ia".

4.4 Vérification Finale

Listez à nouveau les magasins. "cours_ia" ne doit plus apparaître.

Show the list of file search stores.
