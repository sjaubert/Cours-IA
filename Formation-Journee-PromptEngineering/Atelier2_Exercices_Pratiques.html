ATELIER 2 : Constructeur de Prompts - Exercices Pratiques

Durée : 1h00

Objectifs Pédagogiques

À la fin de cet atelier, les participants seront capables de :

  • Construire un prompt structuré de A à Z
  • Adapter un prompt selon le contexte
  • Évaluer et améliorer un prompt existant
  • Utiliser le template universel en autonomie

Introduction (5 min)

Rappel de l'Atelier 1 :

  • Rapide tour de table : "Quel pilier vous semble le plus important ?"
  • Présentation de l'objectif : "Vous allez créer 5 prompts de A à Z"

Présentation du Constructeur Interactif :

  • Si version web disponible : démonstration live
  • Sinon : utilisation du template papier

EXERCICE 1 - Niveau Débutant : "Email Simple" (10 min)

Scénario

Vous devez informer votre équipe d'un changement dans les horaires de la réunion hebdomadaire.

Instructions

Utilisez le template pour créer un prompt qui génère cet email.

Template à Remplir

RÔLE & CONTEXTE :
_____________________________________________

OBJECTIF :
_____________________________________________

TÂCHE :
_____________________________________________

CONTRAINTES :
_____________________________________________

FORMAT :
_____________________________________________

Corrigé Proposé

RÔLE & CONTEXTE :
Tu es un manager d'équipe dans une entreprise industrielle.

OBJECTIF :
Informer l'équipe du changement d'horaire de la réunion 
hebdomadaire sans créer de confusion.

TÂCHE :
Rédige un email professionnel et clair.

CONTRAINTES :
- Ton professionnel mais chaleureux
- Maximum 150 mots
- Préciser l'ancien ET le nouvel horaire
- Donner la raison du changement

FORMAT :
Email classique avec :
- Objet clair
- Salutation
- Corps en 3 paragraphes (raison, changement, action)
- Formule de politesse

Mise en Commun (5 min)

  • 2-3 volontaires lisent leur prompt
  • Le groupe identifie les points forts et ce qui peut être amélioré
  • Le formateur présente le corrigé

EXERCICE 2 - Niveau Intermédiaire : "Procédure Qualité" (12 min)

Scénario

Vous travaillez dans le service qualité. Suite à plusieurs non-conformités sur une ligne de production, vous devez créer une procédure de contrôle visuel pour les opérateurs.

Contexte Complémentaire

  • Les opérateurs ont des niveaux de formation variés (CAP à BAC+2)
  • La procédure doit être affichée au poste de travail (format A4)
  • Elle porte sur le contrôle des dimensions d'une pièce usinée
  • Temps de contrôle maximum : 2 minutes par pièce

Questions Guides

Avant de commencer, répondez à ces questions :

  1. Quel rôle donner à l'IA ? (Expert qualité ? Formateur ?)
  2. Quel est l'objectif principal ? (Clarté ? Exhaustivité ? Rapidité ?)
  3. Quels formats sont adaptés ? (Checklist ? Schéma commenté ?)
  4. Quelles contraintes techniques ? (Mesures, tolérances, outils)

Corrigé Proposé

RÔLE & CONTEXTE :
Tu es un ingénieur qualité dans l'industrie automobile avec 10 ans d'expérience.
Tu t'adresses à des opérateurs de production de niveau CAP à BAC+2.

OBJECTIF :
Permettre aux opérateurs de réaliser un contrôle visuel et dimensionnel 
fiable en moins de 2 minutes, réduisant ainsi les non-conformités.

TÂCHE :
Crée une procédure de contrôle visuel pour une pièce usinée.

CONTRAINTES :
- Format A4 imprimable
- Langage simple, pas de jargon technique
- Intégrer les points de contrôle : dimensions, aspect surface, bavures
- Temps d'exécution : maximum 2 minutes
- Référencer les outils : pied à coulisse, jauge de profondeur
- Inclure les tolérances : ±0.1mm

FORMAT :
Tableau en 4 colonnes :
| Étape | Point de Contrôle | Outil | Critère OK/NOK |
Ajouter une section "Que faire en cas de NOK ?"

ÉTAPES :
1. Liste d'abord les 5-6 points critiques à contrôler
2. Organise-les dans l'ordre logique de contrôle
3. Formule les critères de manière binaire (OK/NOK)
4. Ajoute les actions en cas de non-conformité

EXERCICE 3 - Niveau Intermédiaire+ : "Formation Interactive" (15 min)

Scénario

Vous êtes formateur et vous devez créer un quiz interactif de 10 questions pour évaluer la compréhension d'un module de formation sur "Les bases de la cybersécurité".

Contexte

  • Public : employés de bureau, tous niveaux (administratifs, commerciaux, RH)
  • Objectif du quiz : sensibiliser et évaluer les réflexes de sécurité
  • Format souhaité : QCM avec 4 réponses possibles
  • Niveau de difficulté : accessible mais pas trop simple
  • Thèmes à couvrir : mots de passe, phishing, mises à jour, données sensibles

Instructions par Étapes

ÉTAPE 1 (5 min) : Rédigez votre prompt complet

ÉTAPE 2 (5 min) : Échangez votre prompt avec votre voisin

  • Jouez le rôle de l'IA : qu'est-ce qui n'est pas clair ?
  • Donnez 2 suggestions d'amélioration

ÉTAPE 3 (5 min) : Améliorez votre prompt selon le feedback reçu

Grille d'Auto-Évaluation

Avant l'échange :

  • Rôle précis (expert cybersécurité ? formateur ?)
  • Objectif clair (sensibiliser / évaluer / les deux ?)
  • Contraintes de difficulté spécifiées
  • Format exact du QCM précisé
  • Équilibre entre les thèmes demandé
  • Ton adapté au public (pas trop technique)

Point Pédagogique Fort

L'exercice 3 avec peer review est souvent le déclic : les participants réalisent que leur prompt "évident" ne l'est pas pour les autres. C'est le moment pédagogique le plus fort de l'atelier.

EXERCICE 4 - Niveau Avancé : "Rapport de Synthèse" (15 min)

Scénario Complexe

Vous êtes responsable de production. Vous avez participé à une réunion de 2h sur l'amélioration continue. Vous avez des notes désorganisées. Vous devez créer un prompt pour que l'IA vous aide à produire un rapport de synthèse pour votre direction.

Vos Notes (Fournies aux Participants)

- Problème ligne 3 : cadence trop lente (actuellement 45 pièces/h, objectif 60)
- Jean propose de réorganiser le poste d'emballage
- Coût estimé : 15K€ 
- Marie pense qu'il faut d'abord former les nouveaux
- Le budget formation est serré cette année
- Délai de mise en œuvre si on valide : 3 mois
- ROI estimé : 12 mois selon les calculs de Sophie
- Le directeur veut une réponse avant fin de semaine
- Points positifs : équipe motivée, bon esprit
- Risques : période de forte charge en juin (mauvais timing ?)
- Alternative proposée : investir dans un automate (50K€, ROI 18 mois)

Défis Spécifiques

  • Données brutes : L'IA doit trier et organiser
  • Public exigeant : La direction veut du concis et du factuel
  • Décision attendue : Le rapport doit faciliter la décision
  • Objectivité : Présenter les pour/contre sans biais

Éléments Clés à Intégrer

  • Rôle adapté (manager ? analyste ?)
  • Objectif clair (informer vs convaincre vs décider ?)
  • Structure du rapport définie (sections ? longueur ?)
  • Ton approprié pour la direction
  • Traitement des données brutes expliqué
  • Éléments obligatoires : contexte, enjeux chiffrés, options, recommandation
  • Contraintes de longueur (direction = pas de temps)

Points Clés pour le Formateur

Le formateur présente le corrigé et insiste sur :

  • L'importance du format (tableau = lecture rapide)
  • Le ton neutre (direction n'aime pas qu'on décide à sa place)
  • Les chiffres (toujours chiffrer pour la direction)

EXERCICE 5 - Challenge Final : "Créez Votre Propre Cas" (10 min)

Maintenant, c'est à vous ! Créez un prompt pour votre propre besoin professionnel.

Étapes

  1. Identifiez une tâche que vous répétez souvent (5 min)
    • Exemples : Rédaction de comptes-rendus, Préparation de supports de formation, Réponses à des demandes clients, Analyses de données, Création de checklist
  2. Rédigez le prompt complet avec les 6 piliers (5 min)
  3. Testez-le en conditions réelles (après la formation, en autonomie)

Grille d'Auto-Évaluation

  • Le prompt se suffit à lui-même (on comprend tout sans contexte supplémentaire)
  • Les 6 piliers sont présents
  • Le format de sortie est précisément défini
  • Les contraintes sont réalistes et utiles
  • Le niveau de langage est adapté au public cible
  • Si tâche complexe, les étapes de raisonnement sont guidées

Bilan de l'Atelier (5 min)

Questions de Réflexion

  • Quel exercice vous a semblé le plus utile ?
  • Quelle difficulté avez-vous rencontrée le plus souvent ?
  • Quel pilier oubliez-vous le plus fréquemment ?

Les 3 Erreurs Fréquentes Observées

  1. Oublier de définir le public cible (impact sur le vocabulaire)
  2. Rester vague sur le format (l'IA devine et se trompe souvent)
  3. Ne pas donner d'exemple pour les formats complexes

Les 3 Bonnes Pratiques à Retenir

  1. Toujours commencer par le template (structure mentale)
  2. Se mettre à la place de l'IA : "Est-ce que j'ai TOUT pour répondre ?"
  3. Tester et itérer : le premier prompt n'est jamais parfait

Bibliothèque de Prompts Starter

5 Prompts Prêts à l'Emploi remis aux participants :

1. Email Professionnel

Rédige un email [ton : formel/chaleureux/neutre] pour [objectif] 
destiné à [public]. Longueur max [X] mots.
Structure : objet + salutation + 2-3 paragraphes + formule de politesse.

2. Synthèse de Document

Tu es un analyste. Synthétise ce document en [X] mots maximum.
Public cible : [qui ?]. 
Format : [liste à puces / tableau / paragraphe].
Mets en avant : [éléments clés à extraire].

3. Checklist / Procédure

Crée une checklist pour [tâche] destinée à [public avec niveau].
Format : tableau [étape | action | critère de validation].
Contraintes : [temps / outils / niveau de détail].

4. Support de Formation

Tu es un formateur expert en [domaine].
Crée [type de support] sur [sujet] pour des apprenants [niveau].
Durée : [X] minutes. Inclus : objectifs pédagogiques, exercices, évaluation.
Ton : pédagogique et encourageant.

5. Analyse de Données

Analyse ce jeu de données : [décrire ou copier-coller].
Objectif : [identifier tendances / trouver anomalies / comparer périodes].
Format de sortie : 
1. Statistiques clés (tableau)
2. 3 observations principales
3. 1 recommandation d'action.
