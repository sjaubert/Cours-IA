Jour 2 : Module 7 (14h00 - 16h00)

L'IA Avancée – Workflows et Agents Autonomes

Pôle Formation UIMM-CVDL (S. JAUBERT)

Objectifs Pédagogiques

  • Différencier l'agence conversationnelle (un chat) de l'agence par workflow (un agent qui agit).
  • Comprendre le RAG (Retrieval-Augmented Generation) comme un outil fondamental d'intégration.
  • Appréhender comment l'IA peut utiliser des "Outils" (Tools) pour interagir avec des systèmes externes (API, Web).

Contenu du Cours

1. Le RAG (Retrieval-Augmented Generation)

Le RAG est la technique qui permet à une IA de répondre à des questions en se basant sur une base de connaissances privée (vos documents).

Processus RAG :
Utilisateur pose une question ➔ L'IA cherche la réponse dans la base de documents (PDFs, etc.) ➔ L'IA utilise les extraits trouvés comme "Contexte" ➔ L'IA génère la réponse.

C'est la solution la plus rapide pour déployer une IA qui connaît vos données internes sans avoir à la "fine-tuner" (ce qui est long et coûteux).

2. Agence Conversationnelle vs. Workflow

  • Agence Conversationnelle (Le "Chat") : L'IA est très générale, elle peut parler de tout, mais elle ne peut *rien faire*. Elle est forte en généralité, mais faible en force de frappe.
  • Agence par Workflow (L'"Agent") : L'IA est spécialisée dans une tâche complexe (ex: "gérer le marketing"). Elle est moins générale, mais très forte dans son domaine. Elle décompose un gros objectif en petites tâches.

3. Les "Outils" (Tools) : Donner des mains à l'IA

La vraie puissance des agents vient de leur capacité à utiliser des "Outils". Un outil est une fonction que l'IA peut appeler.

Exemple de cycle (ReAct) :

  1. Pensée (Thought) : L'IA pense : "L'utilisateur veut connaître la météo à Paris. Je dois utiliser l'outil 'get_weather'".
  2. Action (Act) : L'IA appelle l'outil : `get_weather(city="Paris")`.
  3. Observation (Observe) : Le système exécute l'outil, qui renvoie : "15°C, Pluvieux".
  4. Pensée (Thought) : L'IA pense : "J'ai la réponse."
  5. Réponse : L'IA dit à l'utilisateur : "Il fait 15°C et il pleut à Paris."

Activités Pratiques (1h30)

Activité 1 : Démonstration RAG (15 min)

Objectif : Voir la différence entre une IA "nue" et une IA "augmentée".

Consigne : Le formateur charge le PDF `guide_ia_enseignants_1.pdf` (ou un autre document pertinent) dans un outil RAG (ex: Claude, ou un ChatGPT custom).

  1. Prompt 1 (IA Nue) : 
    Qu'est-ce que le 'mega-prompt ultime' selon le guide de Philippe Piekoszewski-Cuq ?
    (L'IA nue répondra qu'elle ne connaît pas ce guide.)
  2. Prompt 2 (IA Augmentée - RAG) : 
    Qu'est-ce que le 'mega-prompt ultime' selon le guide chargé en contexte ?
    (L'IA (RAG) va trouver la réponse et citer le document.)

Activité 2 : Conception d'un Agent (45 min)

Objectif : Penser en termes de "workflow" et d'"outils".

Consigne : En groupes, concevez sur papier (ou tableau blanc) un "Agent IA" pour une tâche industrielle. Définissez son objectif et les "Outils" (fonctions) dont il aurait besoin.

Exemple 1 (Agent de Maintenance Prédictive) :

  • Objectif : Prévenir les pannes sur la Ligne 4.
  • Outils requis :
    1. `read_sensor_logs(ligne_id)` : Lit les données des capteurs.
    2. `analyze_vibration(data)` : Analyse les anomalies (tâche LLM).
    3. `create_incident_ticket(ligne_id, description)` : Ouvre un ticket dans la GMAO.
    4. `send_email(destinataire, sujet, corps)` : Prévient le chef d'équipe.

Exemple 2 (Agent d'Optimisation de Stock) :

  • Objectif : Éviter les ruptures de stock sur les matières premières.
  • Outils requis :
    1. `get_stock_level(item_sku)` : Vérifie le stock actuel (API de l'ERP).
    2. `get_forecast(item_sku)` : Vérifie les prévisions de vente (API BI).
    3. `calculate_order_need(stock, forecast)` : Calcule le besoin (tâche LLM).
    4. `create_purchase_order(supplier_id, item_sku, quantity)` : Crée un bon de commande.

Activité 3 : Restitution (30 min)

Chaque groupe présente son agent, ses objectifs et les outils qu'il a imaginés.
