Introduction

Bienvenue dans ce cours

Ce que vous allez apprendre, pourquoi c'est utile, et comment aborder ce guide.

Imaginez que vous avez un assistant très capable, mais à qui vous devez réexpliquer vos préférences, vos méthodes de travail et votre vocabulaire métier à chaque nouvelle conversation. Frustrant, non ?

Les Skills (compétences) pour Claude résolvent exactement ce problème. Ce sont des instructions packagées une seule fois, que Claude charge automatiquement quand le besoin se présente. Vous enseignez une fois, vous bénéficiez à chaque utilisation.

Chapitre 1

Les Fondamentaux

Comprendre ce qu'est une skill, comment elle s'organise et les principes qui la rendent efficace.

Qu'est-ce qu'une skill ?

Une skill est un dossier contenant des instructions en Markdown qui apprend à Claude comment gérer des tâches ou des workflows spécifiques. C'est tout. Pas de code complexe obligatoire, pas de serveur à déployer pour commencer.

💡
Analogie culinaire

Si Claude est un chef cuisinier professionnel avec tous les ustensiles (ses capacités natives), une skill est une recette : elle indique exactement quoi faire, dans quel ordre, avec quels ingrédients.

Cas d'usage typiques :

  • Générer des designs frontend depuis des spécifications
  • Conduire des recherches avec une méthodologie constante
  • Créer des documents qui suivent le guide de style de votre équipe
  • Orchestrer des processus en plusieurs étapes
  • Planifier des sprints de projet

Structure d'un dossier de skill

📁 Structure de dossierkebab-case obligatoire
ma-skill-nom/
├── SKILL.md          ← Obligatoire – fichier principal
├── scripts/          ← Optionnel – code exécutable
│   ├── traiter_data.py
│   └── valider.sh
├── references/       ← Optionnel – documentation complémentaire
│   ├── guide-api.md
│   └── exemples/
└── assets/           ← Optionnel – modèles, polices, icônes
    └── modele-rapport.md
⚠️
Règles critiques sur les noms

Le fichier principal doit s'appeler exactement SKILL.md (sensible à la casse). Le dossier doit utiliser le format kebab-case : mon-projet-skill et non Mon Projet Skill ni mon_projet_skill.

Principes de conception

1. Divulgation progressive (Progressive Disclosure)

Les skills utilisent un système à 3 niveaux pour optimiser l'utilisation de la mémoire de Claude :

Niveau 1 – Frontmatter YAML
Toujours chargé
→
Claude sait quand activer la skill
Niveau 2 – Corps SKILL.md
Chargé si pertinent
→
Instructions complètes et guidance
Niveau 3 – Fichiers liés
Chargés si nécessaire
→
Documentation, exemples, références

2. Composabilité

Claude peut charger plusieurs skills simultanément. Votre skill doit fonctionner correctement aux côtés d'autres skills, sans supposer qu'elle est la seule disponible.

3. Portabilité

Les skills fonctionnent identiquement sur Claude.ai, Claude Code et via l'API. Créez-en une, utilisez-la partout.

Chapitre 2

Planification et Conception

Avant d'écrire la moindre ligne, identifiez clairement vos besoins.

Commencer par les cas d'usage

Avant d'écrire quoi que ce soit, identifiez 2 à 3 cas d'usage concrets que votre skill doit couvrir. Un bon cas d'usage répond à ces questions :

🤔
Questions à se poser

• Qu'est-ce que l'utilisateur veut accomplir ?
• Quelles étapes multiples ce processus requiert-il ?
• Quels outils sont nécessaires (natifs ou MCP) ?
• Quel savoir-faire ou quelles bonnes pratiques faut-il intégrer ?

Exemple bien défini : Planification de Sprint

📋 Cas d'usage
Cas d'usage : Planification de sprint projet
Déclencheur : L'utilisateur dit "aide-moi à planifier ce sprint"
              ou "crée les tâches du sprint"

Étapes :
  1. Récupérer l'état actuel du projet (via Linear/Jira)
  2. Analyser la vélocité et la capacité de l'équipe
  3. Suggérer une priorisation des tâches
  4. Créer les tâches avec les bons labels et estimations

Résultat attendu : Sprint entièrement planifié, tâches créées

Les 3 catégories de skills

📄 Catégorie 1 – Création de documents
Créer des outputs cohérents et de haute qualité Exemples :
  • Skill docx pour des rapports Word
  • Skill pptx pour des présentations
  • Skill frontend-design pour des interfaces
Techniques clés :
  • Guides de style intégrés
  • Structures de modèles
  • Checklists de qualité
⚙️ Catégorie 2 – Automatisation de workflows
Processus en plusieurs étapes avec méthodologie constante Exemple réel : skill-creator

"Guide interactif pour créer de nouvelles skills."

Techniques clés :
  • Workflow avec portes de validation
  • Modèles pour structures communes
  • Boucles d'amélioration itérative
🔌 Catégorie 3 – Amélioration MCP
Guidance de workflow pour enrichir un serveur MCP Exemple réel : sentry-code-review

"Analyse et corrige automatiquement les bugs dans les Pull Requests via Sentry MCP."

Techniques clés :
  • Coordonne plusieurs appels MCP
  • Expertise métier intégrée
  • Gestion des erreurs MCP courantes

Définir les critères de succès

Comment saurez-vous que votre skill fonctionne ? Fixez des objectifs mesurables avant de commencer :

MétriqueObjectif cibleComment mesurer
Taux de déclenchement ≥ 90% des requêtes pertinentes Tester 10–20 requêtes variées et compter les activations automatiques
Appels API échoués 0 échec par workflow Surveiller les logs du serveur MCP pendant les tests
Autonomie de Claude Pas de relance nécessaire Tester si un nouvel utilisateur réussit du premier coup
Cohérence des résultats Structure identique à chaque fois Répéter la même requête 3–5 fois et comparer
Chapitre 3

Rédiger votre SKILL.md

Le cœur de toute skill : apprendre à structurer le fichier que Claude va lire.

Le frontmatter YAML

Le frontmatter YAML est la partie la plus importante de votre skill. C'est lui qui permet à Claude de décider quand activer la skill, sans charger son contenu complet.

SKILL.mdFormat minimum obligatoire
---
name: mon-skill-nom
description: Ce que ça fait et quand l'utiliser. Inclure des phrases déclencheurs spécifiques.
---

# Le corps de la skill commence ici...

Champs du frontmatter

ChampObligatoire ?Règles
name ✅ Oui Format kebab-case uniquement, pas de majuscules ni espaces. Doit correspondre au nom du dossier.
description ✅ Oui Doit contenir QUOI (ce que ça fait) ET QUAND (phrases déclencheurs). Max 1024 caractères. Pas de balises XML.
license ❌ Non Utile si open-source : MIT, Apache-2.0…
compatibility ❌ Non Environnements requis, packages système, accès réseau. Max 500 caractères.
metadata ❌ Non Paires clé-valeur libres : auteur, version, mcp-server…
🚫
Interdit dans le frontmatter

• Balises XML avec < ou > (risque de sécurité – injection de prompts)
• Noms contenant "claude" ou "anthropic" (réservés par Anthropic)

La description : c'est crucial !

La description est lue en permanence dans le contexte système de Claude. C'est elle qui décide si votre skill sera activée. Rédigez-la avec soin :

❌ Mauvais exemples
description: Aide avec les projets.

# Trop vague
description: Crée des documents sophistiqués.

# Pas de déclencheurs
description: Implémente le modèle d'entité
avec relations hiérarchiques.
# Trop technique, aucun déclencheur utilisateur
✅ Bons exemples
description: Gère les workflows de projet
Linear : planification de sprint, création
de tâches, suivi de statut. Utiliser quand
l'utilisateur mentionne "sprint", "tâches
Linear", "planification projet" ou demande
à "créer des tickets".

description: Analyse les fichiers design
Figma et génère la documentation pour les
développeurs. Utiliser quand l'utilisateur
uploade des fichiers .fig, demande des
"specs design" ou un "handoff design-code".
🧪
Astuce de diagnostic

Demandez à Claude : "Quand utiliseriez-vous la skill [nom] ?" — Claude va citer votre description. Ajustez selon ce qui manque.

Rédiger les instructions

Après le frontmatter, rédigez les instructions en Markdown. Voici la structure recommandée :

SKILL.mdStructure recommandée
---
name: mon-workflow
description: [...]
---

# Ma Skill

## Instructions

### Étape 1 : [Première étape]
Explication claire de ce qui se passe.

### Étape 2 : [Deuxième étape]
Lancer `python scripts/valider.py --input {fichier}` pour vérifier.
Si la validation échoue, problèmes courants :
- Champs obligatoires manquants
- Format de date incorrect (utiliser AAAA-MM-JJ)

## Exemples

### Exemple 1 : [Scénario courant]
L'utilisateur dit : "Configurer un nouvel espace marketing"

Actions :
1. Récupérer les projets existants via MCP
2. Créer un nouveau projet avec les paramètres fournis

Résultat : Projet créé avec lien de confirmation

## Résolution de problèmes

### Erreur : [Message d'erreur courant]
Cause : [Pourquoi ça arrive]
Solution : [Comment corriger]

Bonnes pratiques pour les instructions

❌ Trop vague
Validez les données avant
de continuer.
✅ Précis et actionnable
Lancer `python scripts/validate.py
--input {filename}` pour vérifier.
Si échec, problèmes courants :
- Champs manquants (à ajouter dans le CSV)
- Formats de date invalides (AAAA-MM-JJ)

Exemples complets de frontmatter

Exemple 1 – Skill minimaliste
---
name: redaction-compte-rendu
description: Rédige un compte-rendu de réunion structuré.
  Utiliser quand l'utilisateur dit "rédige le compte-rendu",
  "génère le CR de la réunion" ou "résume les points de la réunion".
---
Exemple 2 – Skill complète avec métadonnées
---
name: onboarding-client-payflow
description: Workflow complet d'onboarding client pour PayFlow.
  Gère la création de compte, la configuration du paiement et la
  gestion des abonnements. Utiliser quand l'utilisateur dit
  "nouveau client", "configurer abonnement" ou "créer compte PayFlow".
license: MIT
compatibility: Nécessite le serveur MCP PayFlow connecté.
  Testé sur Claude.ai et Claude Code.

Chapitre 4

Tester et Itérer

Une skill non testée est une skill qui échouera au mauvais moment. Voici comment valider.

Les 3 types de tests essentiels

1. Tests de déclenchement

Objectif : S'assurer que la skill se charge au bon moment (ni trop souvent, ni pas assez).

Exemple de suite de tests de déclenchement
# ✅ Doit déclencher la skill :
- "Aide-moi à configurer un nouvel espace ProjectHub"
- "Je dois créer un projet dans ProjectHub"
- "Initialiser un projet ProjectHub pour le Q4"

# ❌ Ne doit PAS déclencher la skill :
- "Quelle est la météo à Paris ?"
- "Aide-moi à écrire du code Python"
- "Crée un tableur" (sauf si la skill gère les tableurs)

2. Tests fonctionnels

Objectif : Vérifier que la skill produit les bons outputs.

Structure d'un test fonctionnel
Test : Créer un projet avec 5 tâches
Données : Nom "Planning Q4", 5 descriptions de tâches

Quand : La skill exécute le workflow
Alors :
  - Projet créé dans ProjectHub
  - 5 tâches créées avec les bonnes propriétés
  - Toutes les tâches liées au projet
  - Aucune erreur API

3. Comparaison de performance

Objectif : Prouver que la skill améliore les résultats.

MesureSans skillAvec skill
Instructions par session Répétées à chaque fois 0 (automatique)
Messages aller-retour 15 échanges 2 questions de clarification
Appels API échoués 3 échecs avec retry 0 échec
Tokens consommés ~12 000 ~6 000

Stratégie d'itération

🎯
Pro tip : Itérez sur une tâche à la fois

Les meilleurs créateurs de skills itèrent sur une seule tâche difficile jusqu'à ce que Claude réussisse, puis extraient cette approche dans une skill. C'est beaucoup plus rapide que de tester un large éventail de cas dès le départ.

Signaux d'alerte à surveiller

⬇️ Sous-déclenchement
Symptômes :
- La skill ne se charge pas
- Les utilisateurs l'activent manuellement
- Questions support "quand utiliser X ?"

Solution : Ajoutez plus de phrases
déclencheurs, surtout les termes
techniques et les synonymes.
⬆️ Sur-déclenchement
Symptômes :
- Skill chargée pour des sujets non liés
- Les utilisateurs la désactivent
- Confusion sur son périmètre

Solution : Ajoutez des déclencheurs
négatifs, précisez le périmètre dans
la description.
Exemple de déclencheur négatif
description: Analyse statistique avancée de fichiers CSV.
  Utiliser pour la modélisation statistique, la régression, le clustering.
  NE PAS utiliser pour l'exploration de données simple (utiliser la skill
  data-viz à la place).

Comment distribuer une skill

1
Télécharger le dossier de skill

Cloner le repo GitHub ou télécharger le ZIP de la skill.

2
Installer dans Claude

Ouvrir Claude.ai → Paramètres → Capacités → Skills → Uploader la skill (dossier zippé).

3
Activer la skill

Basculer l'interrupteur de la skill et vérifier que le serveur MCP est bien connecté si nécessaire.

4
Tester

Demander à Claude : "Configure un nouveau projet dans [Votre Service]" et vérifier que la skill s'active.

Patterns avancés

Pattern 1 – Orchestration séquentielle

Idéal quand vos utilisateurs ont besoin de processus en plusieurs étapes dans un ordre précis.

Exemple : Onboarding client
## Workflow : Onboarder un Nouveau Client

### Étape 1 : Créer le compte
Appel MCP : `create_customer`
Paramètres : nom, email, entreprise

### Étape 2 : Configurer le paiement
Appel MCP : `setup_payment_method`
Attendre : vérification du moyen de paiement

### Étape 3 : Créer l'abonnement
Appel MCP : `create_subscription`
Paramètres : plan_id, customer_id (de l'Étape 1)

### Étape 4 : Envoyer l'email de bienvenue
Appel MCP : `send_email`
Modèle : welcome_email_template

Pattern 2 – Coordination Multi-MCP

Pour les workflows qui couvrent plusieurs services différents.

Exemple : Handoff Design → Développement
### Phase 1 : Export Design (Figma MCP)
1. Exporter les assets design depuis Figma
2. Générer les spécifications design

### Phase 2 : Stockage (Drive MCP)
1. Créer le dossier projet dans Drive
2. Uploader tous les assets

### Phase 3 : Création de tâches (Linear MCP)
1. Créer les tâches de développement
2. Attacher les liens d'assets aux tâches

### Phase 4 : Notification (Slack MCP)
1. Poster le résumé dans #engineering

Pattern 3 – Intelligence Métier Spécialisée

Quand votre skill apporte de l'expertise au-delà du simple accès aux outils.

Exemple : Conformité paiements
## Traitement de Paiement avec Conformité

### Avant le Traitement (Vérification Conformité)
1. Récupérer les détails de la transaction via MCP
2. Appliquer les règles de conformité :
   - Vérifier les listes de sanctions
   - Vérifier les autorisations de juridiction
   - Évaluer le niveau de risque
3. Documenter la décision de conformité

### Traitement
SI conformité validée :
  - Appeler l'outil de traitement MCP
  - Appliquer les vérifications anti-fraude appropriées
SINON :
  - Signaler pour révision manuelle
  - Créer un dossier de conformité
Chapitre 6

Résolution de Problèmes

Les erreurs courantes et comment les corriger rapidement.

Problèmes courants

🔴 La skill ne s'uploade pas

❗
Erreur : "Could not find SKILL.md in uploaded folder"

Cause : Le fichier ne s'appelle pas exactement SKILL.md.
Solution : Renommer en SKILL.md (sensible à la casse). Vérifier avec ls -la que le fichier est bien nommé.

❗
Erreur : "Invalid frontmatter" ou "Invalid skill name"

Causes fréquentes :

❌ Mauvais
name: My Cool Skill
# espaces et majuscules

description: "Fait des choses
# guillemet non fermé

name: my-skill
description: Fait des choses
# délimiteurs --- manquants
✅ Correct
---
name: my-cool-skill
description: Fait des choses.
  Utiliser quand...
---

🟡 La skill ne se déclenche pas

⚠️
Symptôme : La skill ne se charge jamais automatiquement

Solutions :
• La description est-elle trop générique ? ("Aide avec les projets" ne fonctionnera pas)
• Inclut-elle des phrases déclencheurs que les utilisateurs prononceraient vraiment ?
• Mentionne-t-elle les types de fichiers si pertinent ?

Diagnostic rapide : Demandez à Claude "Quand utiliseriez-vous la skill [nom] ?" — il va citer votre description et vous indiquer ce qui manque.

🟠 Les instructions ne sont pas suivies

⚠️
Symptôme : Claude charge la skill mais ne suit pas les instructions

Causes courantes :

  • Instructions trop longues → Raccourcissez, utilisez des listes et des numéros. Déplacez les détails dans references/.
  • Instructions enfouies → Mettez les points critiques en haut. Utilisez ## IMPORTANT ou ## CRITIQUE.
  • Langage ambigu → Remplacez "valider correctement" par "CRITIQUE : avant d'appeler create_project, vérifier que le nom n'est pas vide et qu'au moins un membre est assigné."
  • "Paresse" du modèle → Ajoutez : "Prenez le temps de faire cela minutieusement. La qualité prime sur la vitesse."

🔵 Problèmes de connexion MCP

🔌
Symptôme : La skill se charge mais les appels MCP échouent

Checklist :
1. Vérifier que le serveur MCP est connecté : Paramètres → Extensions → [Votre Service] (doit afficher "Connecté")
2. Vérifier l'authentification : clés API valides, permissions correctes, tokens OAuth rafraîchis
3. Tester le MCP indépendamment : "Utilise le MCP [Service] pour récupérer mes projets" (sans la skill)
4. Vérifier les noms d'outils (sensibles à la casse) dans la documentation de votre serveur MCP
