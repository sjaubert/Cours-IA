Construire un agent IA avec Claude

4 étapes progressives — Cas d'usage professionnels et administratifs

⏱ 6 semaines 🎯 4 étapes 👤 Public non-développeur 📅 Mai 2026

🎯 À l'issue de cette formation, vous saurez…

Créer, tester et améliorer un agent IA professionnel capable d'effectuer des tâches automatisées (veille réglementaire, rédaction de courriers, gestion de documents) — sans avoir besoin d'être développeur.

  • Appeler l'intelligence artificielle Claude depuis une interface simple
  • Rédiger des instructions précises qui guident l'agent efficacement
  • Connecter l'agent à vos outils métier (Drive, Calendar, messagerie)
  • Mesurer et améliorer les résultats de manière rigoureuse

📖 Lexique — Termes techniques expliqués

Référez-vous à ce lexique chaque fois qu'un terme vous semble obscur.

API Interface de communication entre deux logiciels. C'est ce qui permet à votre outil de « parler » à Claude.
Clé API Mot de passe personnel qui autorise votre outil à utiliser Claude. Ne jamais la partager ni la mettre dans un e-mail.
MCP Model Context Protocol — protocole qui connecte Claude à vos outils (Drive, Calendar, Gmail…). C'est le « câble » entre l'agent et vos applications.
Variable d'environnement Information secrète stockée dans votre système, hors du code. Permet de garder la clé API invisible dans le fichier.
Prompt Instruction ou question donnée à l'agent. Un prompt système est l'instruction permanente qui définit son rôle.
Parsing Extraction et lecture structurée d'une réponse. Ex : isoler uniquement le texte parmi d'autres données renvoyées.
Cron Système de planification automatique. Ex : « lance cette action tous les matins à 8h ».
Hallucination Quand l'IA invente une information qui n'existe pas (chiffre, date, référence légale…). L'erreur la plus critique.
Artefact Dans Claude.ai, mini-application générée et exécutée directement dans la conversation, sans installation.

1 Les quatre étapes de la formation

01

Agent mono-outil — Recherche web + résumé automatique

⏱ Semaines 1–2 Débutant 🎓 Exercices : 45 min

Vous construisez votre premier agent IA capable de chercher sur le web et de résumer automatiquement les résultats. L'agent utilise un seul outil : la recherche web.

📌 Appeler Claude via une interface simple 📌 Écrire des instructions efficaces 📌 Lire la réponse de l'agent 📌 Gérer les erreurs courantes

💼 Cas d'usage professionnel — Veille réglementaire RH

Une PME souhaite automatiser sa veille sur le droit du travail. L'agent consulte des sources officielles (Légifrance, service-public.fr) et produit un résumé en 3 points avec les actions à prendre.

Exemple d'instructions données à l'agent (prompt système) :

Tu es un assistant juridique RH spécialisé dans le droit du travail français.
Ton rôle est de surveiller les évolutions réglementaires pour une PME de 50 salariés.

Pour chaque recherche, tu dois :
1. Identifier les sources officielles (Légifrance, service-public.fr, URSSAF)
2. Résumer le changement en 3 points maximum, en langage accessible
3. Indiquer la date d'entrée en vigueur et les actions à prendre
4. Préciser si tu n'es pas certain d'une information

Ne jamais inventer de références légales. Si tu n'es pas sûr, dis-le.

Autres cas d'usage possibles à ce niveau :

  • Administratif : Synthèse quotidienne du Journal Officiel
  • Comptabilité : Veille sur les taux TVA et seuils fiscaux
  • Achats : Résumé des actualités sur les fournisseurs clés
📝
Exercice pratique : Tester l'agent avec ces 3 questions et comparer les réponses :
1. « Quelles sont les règles du temps de travail pour les cadres en 2025 ? »
2. « Montant du SMIC en vigueur ? »
3. « Conditions pour bénéficier du dispositif FNE-Formation ? »
02

Tests avec critères précis et mesurables

⏱ Semaines 2–3 Intermédiaire 🎓 Exercices : 1h

Vous apprenez à mesurer objectivement la qualité des réponses de votre agent. Pas d'instinct : des critères clairs, des tests répétables, et des objectifs chiffrés.

📌 Définir des critères de réussite 📌 Concevoir des scénarios de test 📌 Identifier les réponses hors périmètre 📌 Détecter les inventions de l'agent

Objectifs de qualité à viser :

Ce qu'on mesureCible recommandée
Pertinence des réponses (évaluation humaine)> 85 %
Temps de réponse moyen< 3 secondes
Réponses hors sujet< 5 %
Informations inventées par l'agent0 % toléré

💼 Cas d'usage professionnel — Rédaction de courriers administratifs

Un agent qui rédige des courriers officiels pour une collectivité locale. Chaque courrier est évalué selon 5 critères : formule de politesse, structure, présence de marqueurs [À COMPLÉTER], longueur (150–250 mots), ton formel.

🚨
Signaux d'alerte à surveiller :
• Score < 70 % → retravailler les instructions
• Aucun [À COMPLÉTER] sur des informations manquantes → risque d'invention
• Réponses quasi-identiques à des questions différentes → manque de contexte
03

Ajout progressif d'outils et mémoire conversationnelle

⏱ Semaines 3–5 Avancé 🎓 Exercices : 2 × 1h

Vous connectez progressivement votre agent à vos outils du quotidien (stockage de documents, agenda, messagerie) et lui donnez une mémoire pour se souvenir des échanges précédents.

📌 Connecter Drive / Calendar / Gmail 📌 Définir l'ordre de priorité des outils 📌 Activer la mémoire conversationnelle 📌 Sécuriser les actions irréversibles

Programme semaine par semaine :

  • Semaine 3 : Google Drive uniquement. 5 tests de recherche de documents.
  • Semaine 3 (fin) : Ajouter Google Calendar. Vérifier que l'agent choisit le bon outil.
  • Semaine 4 : Gmail en lecture seule. Pas d'envoi automatique.
  • Semaine 4 (fin) : Envoi d'e-mails uniquement après confirmation explicite.
  • Semaine 5 : Mémoire conversationnelle — l'agent se souvient des échanges précédents.

💼 Cas d'usage — Assistant RH multi-outils

L'agent gère plusieurs actions en séquence : chercher un contrat dans Drive, planifier un entretien dans l'agenda, puis envoyer une convocation par mail — mais jamais sans confirmation pour les actions irréversibles.

Instructions données à l'agent (extrait) :

Tu es l'assistant RH de l'entreprise.

Ordre de priorité des outils :
1. Drive → si la demande porte sur un document, un contrat, une politique RH
2. Calendar → si la demande porte sur des dates ou des disponibilités
3. Gmail → uniquement si demandé explicitement, après confirmation

Règles de sécurité :
— Aucune action irréversible sans confirmation explicite
— Indiquer quelle source tu consultes à chaque étape
— Si l'information est introuvable, le dire clairement
⚙️
Note MCP — connexion aux outils : Les exemples de code de cette étape utilisent des adresses de serveurs illustratives. En pratique, les URLs réelles dépendent de votre déploiement MCP. Contactez votre administrateur système ou consultez la documentation de votre fournisseur pour obtenir les bonnes adresses avant de tester.
📝
Tests multi-outils à exécuter :
Q1 (→ Drive) : « Quelle est notre politique de télétravail ? »
Q2 (→ Calendar) : « Quand a lieu le prochain CODIR ? »
Q3 (ambiguë) : « Prépare la réunion RH de vendredi. »
Attendu sur Q3 : l'agent demande une clarification (créer l'événement ? envoyer les convocations ? rédiger l'ordre du jour ?)
04

Évaluation des erreurs et affinage des instructions

⏱ Semaines 5–6 Expert 🎓 Exercices : 1h30

Vous apprenez à diagnostiquer et corriger les erreurs typiques d'un agent IA, et à affiner ses instructions de manière méthodique jusqu'à obtenir un résultat fiable.

📌 Identifier les 4 erreurs fréquentes 📌 Corriger via les instructions 📌 Itérer selon un cycle structuré 📌 Valider avant mise en production

Les 4 erreurs les plus fréquentes — et leurs corrections :

🎭 Erreur 1 — Hallucination

L'agent invente des chiffres ou des références légales qui n'existent pas.

→ Correction : lui demander d'écrire [DONNÉE MANQUANTE] plutôt que d'estimer.

🔀 Erreur 2 — Mauvais outil

L'agent cherche dans la messagerie au lieu du Drive, ou inversement.

→ Correction : définir clairement l'ordre de priorité des outils dans les instructions.

🔁 Erreur 3 — Boucle infinie

L'agent tourne en rond et ne trouve pas de réponse, sans jamais s'arrêter.

→ Correction : fixer un nombre maximum d'essais (ex : 5 tentatives maximum).

🚫 Erreur 4 — Hors périmètre

L'agent répond à des questions qui ne le concernent pas.

→ Correction : définir explicitement son domaine et la formule de refus poli.

Illustration — Évolution des instructions d'un agent comptable :

VERSION 1 — trop vague ❌

Tu es un assistant comptable.
Aide à gérer les factures.

VERSION 3 — après corrections ✅

Tu es un assistant comptable pour une TPE.

— Montants : toujours 2 décimales (ex: 1 234,56 €)
— Facture ambiguë → demander clarification AVANT
— Distinguer HT / TTC / TVA dans chaque réponse
— Montant absent → écrire MONTANT MANQUANT
— Toujours rappeler le numéro de facture et la date

Cycle d'amélioration continue — en 4 étapes :

🔍
Observer

Collecter les réponses insatisfaisantes

🧭
Orienter

Catégoriser le type d'erreur

⚡
Décider

Prioriser le correctif le plus urgent

✏️
Agir

Modifier les instructions et relancer les tests

✅ Checklist de validation avant mise en production

Score moyen > 85 % sur la batterie de tests
Zéro information inventée sur 20 tests variés
L'agent demande une clarification face aux questions ambiguës
L'agent refuse d'agir sans confirmation sur les actions irréversibles
Le nombre maximum de tentatives est configuré
Le périmètre est respecté (tests avec questions hors domaine)
La mémoire fonctionne sur au moins 5 échanges
Les erreurs sont enregistrées et catégorisées automatiquement
