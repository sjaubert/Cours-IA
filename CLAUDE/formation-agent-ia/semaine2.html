Premier agent HTML avec recherche web

Construire un agent fonctionnel dans un fichier HTML — aucune installation requise

⏱ 2 h 30 en présentiel 👥 Profils mixtes 🔵 Niveau B — Fichier HTML local

🎓 Guide formateur

📋 Déroulé de la séance (2 h 30)
  • 0:00 – 0:20
    Rappel semaine 1 + introduction niveau B (20 min) Question d'ouverture : « Avez-vous utilisé Claude dans la semaine ? Qu'est-ce qui a bien marché ? ». Expliquez la différence entre niveau A (navigateur) et niveau B (fichier HTML avec clé API). Montrez que le fichier agent.html s'ouvre comme n'importe quel fichier — pas de serveur, pas d'installation.
  • 0:20 – 0:50
    Démo formateur : anatomie du fichier agent.html (30 min) Ouvrez le fichier agent_base.html (fourni dans les ressources) sur le vidéoprojecteur. Expliquez les 4 zones : saisie de la clé API, zone de prompt, bouton « Envoyer », zone de réponse. Montrez comment la clé API est stockée localement dans le navigateur (localStorage) — elle ne part jamais sur internet sauf vers Anthropic.
  • 0:50 – 1:35
    Activité 1 : personnaliser son agent avec un prompt système (45 min) Chaque stagiaire ouvre le fichier sur son poste, saisit sa clé API, et modifie le prompt système pour son métier. Le formateur circule.
  • 1:35 – 1:45
    Pause (10 min)
  • 1:45 – 2:20
    Activité 2 : activer la recherche web (web_search) (35 min) Montrez comment ajouter l'outil web_search au fichier HTML. Les stagiaires testent avec des requêtes d'actualité professionnelle.
  • 2:20 – 2:30
    Débrief collectif (10 min) Question : « Quelle différence avez-vous notée entre une réponse avec et sans web_search ? »
🔑 Gestion des clés API Idéalement, fournissez une clé API de test avec un budget limité (5-10 $) pour la formation. Chaque stagiaire peut aussi créer son propre compte Anthropic (console.anthropic.com) et obtenir sa clé gratuitement avec un crédit de départ. La clé est au format sk-ant-api03-….
⚠️ Point de vigilance : sécurité de la clé API Insistez sur ce point : la clé API ne doit jamais être partagée, publiée sur internet ou mise dans un mail. Dans le fichier HTML, elle est stockée dans le navigateur local uniquement. Si un stagiaire la partage par erreur, il peut la révoquer immédiatement sur console.anthropic.com.
🖥️ Matériel nécessaire Fichier agent_base.html distribué sur clé USB ou partagé sur réseau local · Clés API Anthropic (une par stagiaire ou partagée) · Navigateur Chrome ou Firefox récent.

🧩 Les concepts clés de cette semaine

🔑 Clé API
Mot de passe unique qui identifie votre application auprès d'Anthropic. Elle permet au fichier HTML d'envoyer des questions à Claude.
📝 Prompt système
Instruction permanente que Claude reçoit avant votre question. C'est là que vous définissez son rôle, sa personnalité, ses limites.
🌐 web_search
Outil qui permet à Claude de chercher des informations récentes sur internet avant de répondre — comme un moteur de recherche intégré.
📦 localStorage
Espace de stockage dans votre navigateur. La clé API y est sauvegardée localement — elle ne quitte jamais votre ordinateur sauf pour contacter Anthropic.
🔵 Niveau B
Agent dans un fichier HTML local. Plus puissant que le niveau A (accès à l'API, mémoire de conversation), mais sans installation de logiciel.
💬 Messages
L'historique de la conversation envoyé à Claude à chaque message. C'est ce qui lui permet de « se souvenir » des échanges précédents.

🛠️ Activités pratiques

1

Personnaliser son agent avec un prompt système

⏱ 45 min 🔵 Niveau B — Fichier HTML 👤 Individuel
🔵 Fichier agent_base.html

Vous allez ouvrir le fichier agent_base.html fourni par le formateur, saisir votre clé API, puis modifier le prompt système pour créer votre propre assistant métier.

Zone à modifier dans le fichier HTML — prompt système RH

// Trouvez cette ligne dans le fichier et remplacez le texte entre guillemets : const SYSTEM_PROMPT = `Tu es un assistant RH expert spécialisé dans le droit du travail français et la gestion des ressources humaines pour les PME industrielles. Tu réponds toujours en français, de façon précise et professionnelle. Si une information peut avoir évolué récemment (loi, jurisprudence), tu le signales. Tu ne donnes jamais de conseil juridique engageant — tu orientes vers un avocat ou l'inspection du travail pour les cas complexes.`;
💡 En clair, ce que fait ce code Ce texte est une « instruction permanente » envoyée à Claude avant chaque conversation. Imaginez que vous briefez un nouveau collaborateur sur son rôle avant qu'il prenne son premier appel. Claude lira ces instructions à chaque fois que vous lui poserez une question.

Version Comptabilité

const SYSTEM_PROMPT = `Tu es un assistant comptable et fiscal pour PME françaises. Tu maîtrises le plan comptable général, la TVA, les déclarations fiscales courantes et la gestion de trésorerie. Tu réponds en français, avec des exemples concrets. Tu signales toujours si une règle fiscale a pu évoluer et recommandes de vérifier avec l'expert-comptable pour les situations complexes.`;

Version Manager / Production

const SYSTEM_PROMPT = `Tu es un assistant pour responsable d'atelier dans l'industrie manufacturière. Tu maîtrises le lean management, la sécurité au travail, la gestion de production et l'amélioration continue. Tu proposes des solutions pratiques, directement applicables en atelier. Tu utilises un langage simple et opérationnel, sans jargon excessif.`;

📋 Étapes à suivre :

  • 1Ouvrez le fichier agent_base.html dans votre navigateur (double-clic).
  • 2Dans la zone « Clé API », saisissez votre clé (format sk-ant-…) et cliquez « Sauvegarder ».
  • 3Cliquez sur « ⚙️ Modifier le prompt système » et collez le texte correspondant à votre profil.
  • 4Posez 3 questions professionnelles concrètes et évaluez la qualité des réponses.
  • 5Modifiez une phrase du prompt système et observez comment les réponses changent.
2

Activer la recherche web (web_search)

⏱ 35 min 🔵 Niveau B — Fichier HTML 👥 En binôme puis individuel

L'outil web_search permet à Claude d'aller chercher des informations récentes sur internet avant de répondre. Sans cet outil, Claude ne connaît que ce qu'il a appris pendant son entraînement (données jusqu'à début 2025 environ).

Extrait du fichier HTML — activation de web_search

// Dans la configuration de l'appel à l'API Claude : const response = await fetch('https://api.anthropic.com/v1/messages', { method: 'POST', headers: { 'x-api-key': apiKey, 'anthropic-version': '2023-06-01', 'content-type': 'application/json', 'anthropic-beta': 'web-search-2025-03-05' // ← active le web_search }, body: JSON.stringify({ model: 'claude-sonnet-4-6', max_tokens: 2048, tools: [{ type: 'web_search_20250305', name: 'web_search' }], system: SYSTEM_PROMPT, messages: conversationHistory }) });
💡 En clair, ce que fait ce code Ces quelques lignes donnent à Claude un accès à un moteur de recherche. Quand vous posez une question sur l'actualité (nouvelle loi, taux de TVA modifié, indice de prix…), Claude va d'abord chercher sur le web, puis synthétiser les résultats dans sa réponse — comme le ferait un assistant qui vérifie l'information avant de vous répondre.

📋 Questions test à poser à votre agent (avec web_search activé) :

  • 🟢RH : « Quel est le montant du SMIC horaire brut en vigueur aujourd'hui en France ? »
  • 🔵Compta : « Quels sont les taux de TVA applicables en France en 2026 ? »
  • 🟣Manager : « Quelles sont les dernières évolutions réglementaires en matière de sécurité au travail dans l'industrie française ? »
  • +Tous : Posez la même question sans web_search (en commentant la ligne tools) et comparez les deux réponses.

📌 Ce que vous emportez de cette séance

  • Un agent HTML fonctionnel, personnalisé pour votre métier — vous pouvez l'utiliser dès demain
  • web_search = Claude avec accès à l'actualité. Sans web_search = Claude avec ses connaissances jusqu'à début 2025
  • La semaine prochaine : comment mesurer et améliorer la qualité des réponses de votre agent

📦 Fichier fourni cette semaine

Le fichier agent_base.html est distribué par le formateur. Il contient :

  • Interface graphique complète (zone de saisie, zone de réponse)
  • Gestion sécurisée de la clé API dans le navigateur
  • Mémoire de conversation (Claude se souvient de vos messages précédents)
  • Bouton d'activation/désactivation de web_search
  • Paramètre de prompt système modifiable sans toucher au code
💡 Pas besoin de comprendre tout le code Vous n'aurez à modifier que 2 zones dans le fichier : le prompt système (texte) et éventuellement le nom du modèle. Tout le reste fonctionne comme une boîte noire.
