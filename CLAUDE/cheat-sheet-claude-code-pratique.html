Claude Code en pratique : trouver les bons outils et travailler comme un pro

Cheat sheet operationnelle - Formation a l'IA agentique
Pole Formation UIMM-CVDL - S. JAUBERT - Document de reference a conserver pres du poste de travail

A qui s'adresse ce document. A toute personne qui commence a utiliser Claude Code (l'assistant IA qui travaille dans un terminal, c'est-a-dire la fenetre ou l'on tape des commandes texte). Aucune competence informatique avancee n'est requise : chaque astuce est expliquee en langage clair, avec la commande exacte a recopier et un cas d'usage concret.

Comment lire les commandes. Tout ce qui apparait sur fond sombre se tape (ou se copie-colle) dans le terminal, puis on valide avec la touche Entree. Le symbole | (barre verticale) et le symbole > (chevron) sont expliques dans le glossaire en fin de document.

Partie 1 - Trouver le bon outil sans le connaitre par avance

Claude Code peut etre enrichi par des skills (des modules de competences specialises) et des connecteurs MCP (des ponts vers vos logiciels : messagerie, CRM, gestion de projet, etc.). Le probleme n'est pas qu'ils manquent : c'est qu'on ne sait pas lequel installer, ni si on possede deja l'equivalent. D'ou une perte de temps frequente.

A

Installer le seul outil qui trouve tous les autres : find-skills

Une seule installation rend l'ensemble du catalogue consultable. On decrit ensuite ce que l'on veut faire en francais courant, et l'outil propose les modules les plus pertinents, classes par fiabilite.

npx skills add vercel-labs/skills@find-skills

La bonne facon de formuler sa demande : decrire le resultat voulu, pas le nom d'un outil.

A eviter
"trouve-moi un skill Apollo"
Recommande
"j'ai besoin de rechercher des decideurs dans des entreprises financees, par secteur et par fonction"
Cas d'usage formation/industrie : "trouve un module qui transforme un compte rendu de reunion en liste d'actions", ou "je veux extraire les tableaux d'un PDF de specifications". Plus l'intention est precise, plus la reponse est utile.

B

Lire le nombre d'installations avant d'installer

Le catalogue est ouvert : n'importe qui peut publier. Le seul signal honnete de qualite est le nombre de personnes qui ont installe l'outil et l'ont garde.

Nombre d'installationsInterpretation
Moins de 50Peu eprouve. Vous etes proche du statut de testeur. Verifiez le depot avant d'installer.
Plus de 100Des personnes autres que l'auteur l'ont trouve assez utile pour l'adopter. Signal plus rassurant.

Reflexe de verification avant d'installer un module peu telecharge : regarder la date du dernier commit (derniere mise a jour). S'il n'a pas bouge depuis six mois, sans activite ni etoiles, la reponse est non.

Le reflexe a retenir. Avant d'installer quoi que ce soit, demandez a Claude : "ai-je deja de quoi faire cela ?". Bien souvent un connecteur deja en place fait le travail, et une installation supplementaire est inutile. find-skills verifie justement ce que vous possedez deja avant de proposer du neuf.

Partie 2 - Les 13 astuces qui font la difference dans le terminal

Toutes ces astuces reposent sur des principes simples et anciens (les "tuyaux" entre logiciels, la redirection vers un fichier, les raccourcis). Combinees a Claude, elles accelerent fortement le travail. Pour chaque astuce : la facon debutante, la facon pro, et un cas concret.

1

Lancer une commande sans quitter Claude : le !

Debutant
Quitte Claude, ouvre un autre terminal, puis revient expliquer ce qui s'est passe.
Pro
Tape ! suivi de la commande, sans quitter la session.
! npm run build
! git status

Le resultat reste sous les yeux de Claude, qui peut reagir immediatement. On ne perd jamais le fil de la conversation.

Cas d'usage : verifier l'etat d'un dossier ou relancer une operation sans interrompre l'echange en cours.

2

Envoyer une erreur directement a Claude

Debutant
Copie le message d'erreur, le colle dans Claude et ecrit "corrige ca".
Pro
Envoie la sortie d'erreur d'un seul geste.
npm run build 2>&1 | claude -p "corrige ces erreurs"

Claude lit le message exact (et non votre reformulation), avec les numeros de ligne. C'est plus rapide et plus precis.

Cas d'usage : un programme echoue avec un message incomprehensible : on le transmet tel quel a Claude pour diagnostic.

3

Mode rapide (sans conversation) : claude -p

Debutant
Ouvre une session complete meme pour une question d'une ligne.
Pro
Utilise -p pour une reponse unique et immediate.
claude -p "explique ce que fait ce code"
claude -p "genere un fichier .gitignore pour un projet Python"

Le drapeau -p (pour "print", afficher) traite la demande, affiche la reponse et s'arrete. Ideal pour une explication ou une recherche ponctuelle.

Cas d'usage : obtenir une definition, un exemple ou un petit fichier sans lancer tout un echange.

4

Reprendre une session : --resume

Debutant
Recommence de zero et re-explique tout le contexte a chaque fois.
Pro
Reprend la veille exactement la ou il s'etait arrete.
claude --resume

Recharge la derniere conversation : le contexte, les decisions et les fichiers sont toujours la.

Cas d'usage : un dossier traite sur plusieurs jours. Le lendemain, Claude se souvient de ce qui a ete tente et de ce qui a fonctionne.

5

Enchainer Claude avec d'autres outils

Debutant
Lance chaque outil separement et recopie le resultat de l'un dans l'autre.
Pro
Relie les outils en une seule commande.
git diff | claude -p "relis ces modifications"
cat agenda.txt | claude -p "resume les points cles"

Le symbole | (le "tuyau") fait passer le resultat d'un outil dans Claude. Tout ce qui produit du texte peut etre analyse.

Cas d'usage : journaux de bord, sorties de tests, fichiers de configuration, exports... tout texte devient analysable.

6

Creer des raccourcis (alias) pour les taches repetitives

Debutant
Retape la commande complete a chaque fois.
Pro
Cree un raccourci de deux lettres.
# A ajouter une fois dans le fichier ~/.bashrc
alias cr='git diff | claude -p "relis ce code"'
alias cs='claude -p "resume ce projet en 3 phrases"'

Apres rechargement, taper cr suffit. Les gains de temps se cumulent sur des centaines d'usages.

Cas d'usage : automatiser un controle ou un resume que l'on declenche plusieurs fois par jour.

7

Donner les vraies donnees Git a Claude

Debutant
Demande "qu'est-ce que j'ai change ?" et attend.
Pro
Fournit directement l'historique reel.
git log --oneline -10 | claude -p "resume ces changements"

Claude travaille sur les donnees exactes, sans deviner a partir des dates de fichiers.

Cas d'usage : produire un compte rendu fidele des evolutions recentes d'un projet versionne.

8

Integrer Claude dans des scripts automatises

Debutant
Ne sait pas comment utiliser Claude en automatique.
Pro
Utilise --print dans des scripts ou des taches planifiees.
# Controle quotidien planifie (exemple)
claude -p "fais un bilan rapide de ce projet" > bilan.txt

En mode --print, Claude se comporte comme n'importe quel outil : il recoit une entree, produit un texte. On peut donc l'inclure dans une tache recurrente.

Cas d'usage : un bilan automatique chaque matin, ecrit dans un fichier, sans intervention.

9

Lancer une tache longue en arriere-plan : le !

Debutant
Ouvre un autre onglet pour une commande qui tourne longtemps.
Pro
Lance la commande avec ! et continue a discuter.
! npm run test -- --watch

La commande tourne en fond pendant que l'on poursuit l'echange. Claude peut s'y referer ensuite.

Cas d'usage : un serveur de test qui se recharge tout seul pendant que Claude apporte des modifications.

10

Memoriser ses reglages une bonne fois pour toutes

Debutant
Reconfigure sa cle d'acces a chaque nouveau terminal.
Pro
L'enregistre une fois dans son profil et n'y pense plus.
# A ajouter une fois dans ~/.bashrc
alias cc='claude'

Un raccourci de deux lettres economise des milliers de frappes sur un mois.

Cas d'usage : lancer Claude avec cc au lieu de claude a chaque fois.

11

Donner plusieurs instructions en une seule fois

Debutant
Envoie une instruction, attend, puis la suivante.
Pro
Enchaine la sequence complete dans une seule demande.
claude -p "lance les tests, corrige ceux qui echouent,
puis relance jusqu'a ce que tout passe"

Claude execute la suite de maniere autonome : chaque etape nourrit la suivante.

Cas d'usage : deleguer une procedure en plusieurs etapes plutot que de la piloter pas a pas.

12

Enregistrer la reponse de Claude dans un fichier

Debutant
Copie la reponse et la colle manuellement dans un fichier.
Pro
Redirige la sortie vers un fichier d'un seul geste.
claude -p "redige un mode d'emploi du projet" > LISEZMOI.md

Le chevron > ecrit la reponse directement dans le fichier. Avec >>, on ajoute a la suite d'un fichier existant.

Cas d'usage : generer une note, un mode d'emploi ou un modele de document pret a l'emploi.

13

Faire lire les vrais fichiers a Claude : cat

Debutant
Decrit ses fichiers avec des mots.
Pro
Fournit les fichiers reels en entree.
cat notes.txt compte-rendu.txt | claude -p "synthetise ces documents"

Claude lit le contenu reel, sans interpreter une description. Le symbole * permet de transmettre tout un lot de fichiers d'un coup.

Cas d'usage : reunir plusieurs documents pour en faire une synthese unique. A noter : pour de tres gros volumes, preferer la session interactive, ou Claude lit les fichiers de facon selective.

Glossaire express - les symboles a connaitre

Terminal
La fenetre dans laquelle on tape des commandes texte plutot que de cliquer. C'est la qu'on dialogue avec Claude Code.
! (point d'exclamation)
Place devant une commande, il l'execute sans quitter Claude (astuces 1 et 9).
| (barre verticale, "le tuyau")
Fait passer le resultat d'un outil dans un autre. Exemple : git diff | claude -p "..." envoie les modifications a Claude.
> et >> (chevrons)
> ecrit le resultat dans un fichier (en remplacant son contenu). >> ajoute a la fin sans effacer.
2>&1
Capture a la fois les messages normaux et les messages d'erreur, pour tout transmettre a Claude (astuce 2).
-p ou --print
Mode rapide : Claude repond une fois et s'arrete, sans ouvrir de conversation (astuces 3, 8, 12).
alias
Un raccourci personnel : on remplace une commande longue par quelques lettres (astuces 6 et 10).
cat
Commande qui affiche le contenu d'un fichier ; ici, elle sert a transmettre ce contenu a Claude (astuce 13).
skill / connecteur MCP
Un skill est un module de competence ajoute a Claude. Un connecteur MCP est un pont vers un logiciel existant (messagerie, gestion de projet, CRM...).
Le principe a garder en tete. La difference entre un usage debutant et un usage avance ne tient pas a des astuces d'IA, mais a quelques reflexes simples : relier les outils entre eux (le tuyau), enregistrer les resultats (le chevron), creer des raccourcis, et donner a Claude les vraies donnees plutot qu'une description. Commencez par maitriser deux ou trois de ces gestes, le reste viendra naturellement.
